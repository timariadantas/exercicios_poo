# Programação Orientada a Objetos (POO)  Feito em Csharp

A **Programação Orientada a Objetos (POO)** é um paradigma de programação que organiza o código em **objetos**, que representam entidades do mundo real ou conceitos do problema. Cada objeto possui **atributos** (dados) e **métodos** (funções ou comportamentos).

A POO facilita a **manutenção**, **reutilização** e **organização** do código.


## Os 4 Pilares da POO

###  Encapsulamento
- **O que é:** esconder detalhes internos do objeto e controlar o acesso a eles.
expõe apenas o necessário com métodos ou propriedades
obs : os atributos private não pode ser acessado na classe filha , só o método.
- **Como funciona:** usamos **modificadores de acesso** (public, private, protected).

### Herança

- **O que é:** uma classe pode herdar atributos e métodos de outra, evitando repetição de código.

### Polimorfismo “Poli” = muitos “Morfos” = formas

- **O que é:** objetos podem se comportar de formas diferentes, dependendo do contexto.

  ### Abstração

- **que é:** focar apenas no essencial, escondendo detalhes complexos.
Uma classe abstrata é um modelo incompleto.
Ela define o que deve ser feito, mas não o como.
